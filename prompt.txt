Codex report - CI/local parity after Playwright E2E

Implemented CI/local parity improvements.

1. CI/Local Mismatch Found
- CI did not run Playwright E2E.
- `make validate` matched only non-browser CI, but README wording did not make the separate E2E lane clear enough.
- Frontend Prettier checks did not cover `frontend/e2e/**` or `playwright.config.ts`.
- Local `make test-e2e-servers` inherited `DATABASE_URL` from `backend/.env`; in this workspace that pointed at Railway. Fixed so E2E defaults to local Postgres and does not use a real OpenAI key from `.env`.

2. Targets/Scripts Changed
- Added `make quick`: `lint` + backend unit tests.
- Clarified `make validate` / `make ci-local`: non-browser CI mirror.
- Hardened `make test-e2e-servers`:
  - `E2E_DATABASE_URL ?= postgresql://postgres:postgres@localhost:5432/review_insight`
  - `E2E_OPENAI_API_KEY ?= change-me-e2e`
  - backend config clears that placeholder, confirmed with `False scripted True mock`.
- Expanded frontend Prettier scripts to include `e2e/**/*` and `playwright.config.ts`.

3. Playwright In CI
Yes. Added a dedicated `playwright` job in `.github/workflows/ci.yml`:
- Needs backend/frontend jobs.
- Starts Postgres 16 service.
- Installs backend/frontend deps.
- Caches `~/.cache/ms-playwright`.
- Installs Chromium with deps.
- Runs Alembic migrations.
- Starts backend with `TESTING=true`, `LLM_PROVIDER=scripted`, `REVIEW_PROVIDER=mock`, `OPENAI_API_KEY=""`.
- Starts frontend with `NEXT_PUBLIC_API_URL=http://localhost:8000`.
- Runs `npm run test:e2e`.
- Uploads Playwright reports, test results, and server logs on failure.
